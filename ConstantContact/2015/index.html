Types of Attacks

Brute Force


Repeated attacks trying every possible combination until correct code is found.

Dictionary Attack


A guessing attack which runs down a list of options.

SQL Injection


Uses weak code to trick the database into giving information or altering the data in the database.

Code Injection


Exploits a computer bug caused by processing invalid data. Causes the course of execution to change.

Used by computer worms to propagate.

Cross Site Scripting

Allows client side JavaScript execution.

  • Cookie Stealing - use to gain access or bypas security.
  • Cross_Site Request Forgery - makes user send a request unknowingly.
  • XSS Worms - malcious script that can spread to other sites.
  • Keylogging - logs keystrokes.

Protecting Your Site

Use SSL


  • Common in E-commerce
  • Shopping checkout
  • Login pages
  • Data collection
  • Whole site

Data validation


  • Done on server ONLY!
  • NOT client side (JavaScript)

External login

Log in via encrypted connections (SFTP or Open SSH)

Use Secure Networks


Avoid coffeeshop connections, use secured networks

Login Credentials


Don't share login credentials with ANYONE!

Key Access

Try to use key based authentication vs password when available

Secure your Workstation

  • Update the Operating System
  • Update the anti-virus
  • Firewalls enabled
  • Look for suspicious emails
  • Install software from credible sources

Use a non-login email address on public material

Have a specific email address set for promotional information (info@example.com)

Use Strong Passwords

  • Long passwords are good (min 8 characters)
  • Use a 'pass sentence'
  • Avoid the obvious
  • Do not recycle passwords
  • Rotate passwords
  • Use Upper, Lower, letters, and characters

Most common passwords in 2014

Backups! (Part 1)


  • Host copy
  • Developer copy
  • Personal copy
  • Physical copy
  • Digital copy

Backups! (part 2)


  • Files and database
  • Regular intervals
  • Before major changes
  • DONT store backups on the hosting server

Limit Access


Don't give access to everyone.

Give only the access they need.

Stay up to date


Stay up to date on all software versions (both server and client side) includes plugins and CMSes for server and ALL software on the client side

Use caution on Social Media


Watch what you say on your personal social media.

Guard against social engineering


  • Very old method
  • Human hacking
  • Attacks the human tendency to trust
  • Human based
  • Computer based
